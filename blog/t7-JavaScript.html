Hash vs Class: Diaries of JS and Ruby

03/22/2015

If you were thinking hashes in Ruby and Objects in JavaScript look very similar, you're not just imagining it. The syntax of hashes in Ruby, with their key-value pairs and curly braces, mirror the content and sugary syntax within Objects in JavaScript! Take a look at the similar syntax:

A JavaScript object is looks like this --> var car = { type: "Fiat", model: 500, color: "white" };
A Hash in Ruby looks like this --> grades = { "Jane Doe" => 10, "Jim Doe" => 6 }

Here, we can see that in JavaScript you need to use the var keyword when creating a variable, whereas in Ruby you don't need it. The other main difference between the two exists in how the values are accessed.

If you want to access a value in a JS object, then you would do so like this --> car.type which prints out "Fiat"
If you want to access a key's value in a Ruby hash, then you would do so like this --> grades["Jane Doe"] which prints out 10.
